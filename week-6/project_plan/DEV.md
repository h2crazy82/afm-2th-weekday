# DEV — 어떻게 만들 것인가

> 7주차 MVP는 "라라에듀 PM 김지원이 5분 안에 고객사 1곳 맞춤 1차 초안을 받는다"가 합격 기준. 그 외에는 모두 후순위.

## 1. MVP 범위 (Yes/No 명확히)

| 기능 | MVP 포함 | 이유 |
|------|:-------:|------|
| 고객사 정보 입력 폼 (회사명·업종·직원수·위험요소·콘텐츠 종류) | **YES** | 입력 없이는 맞춤 불가 |
| Claude API 기반 채팅형 학습자료 생성 | **YES** | 핵심 기능. 이게 빠지면 의미 없음 |
| 라라에듀 콘텐츠 자산 컨텍스트 주입 (산업안전·법정의무·요즘리더 AI) | **YES** | 차별화의 원천 |
| 채팅 중 "이 부분만 더 디테일하게" 인터랙티브 수정 | **YES** | 일회성 생성기는 GPT랑 차이가 없음 |
| Markdown 다운로드 (.md 파일) | **YES** | 가장 간단한 출력 |
| 고객사별 대화 이력 저장 (Supabase) | **YES** | 재방문 시 컨텍스트 유지 |
| Auth (라라에듀 직원만 로그인) | **YES** | 콘텐츠 자산 노출 방지 |
| **PDF 다운로드** (ReportLab 또는 Puppeteer) | NO | MD → PDF 변환은 8주차 또는 그 이후 |
| 고객사 로고·CI 색상 자동 반영 | NO | 이건 8주차 시연 전 데모용으로만 |
| 평가 문항 5개 자동 생성 | NO | 콘텐츠 본문 → 평가는 별도 기능, 분리 |
| 결제·구독 (라라에듀 외부 고객 SaaS 판매) | NO | 8주차 이후 Phase 2 |
| 다국어 지원 | NO | 한국어 한정 |
| 모바일 최적화 | NO | 데스크톱 사용이 99% (PM이 오피스에서) |

**핵심 3개 기능 (DEV.md 필수)**:
1. **고객사 입력 → 채팅 생성** (10시간 예상)
2. **인터랙티브 수정 + 라라에듀 콘텐츠 자산 RAG** (15시간)
3. **MD 다운로드 + 고객사별 이력** (5시간)

총 ~30시간 / 7주차 (3주 분산)

## 2. 기술 스택

이미 배운 것 + 라라에듀 인프라와 호환되는 것을 우선.

| 레이어 | 선택 | 이유 |
|--------|------|------|
| **Framework** | Next.js 16 App Router | week-6 학습 + 서버 컴포넌트 + Vercel 1-click 배포 |
| **Auth** | Supabase Auth (이메일+비밀번호) | week-5 동일, 라라에듀 직원 7명 가정 |
| **DB** | Supabase Postgres | week-5 프로젝트 재사용. RLS로 본인 회사 데이터만 |
| **AI** | Anthropic Claude API (claude-sonnet-4-6 / claude-haiku-4-5) | 한국어 품질 + 200k 컨텍스트 |
| **RAG** | 단순 system prompt 주입 (벡터 DB 없음) | MVP는 라라에듀 콘텐츠 자산을 system prompt로 prepend |
| **Frontend UI** | shadcn/ui + Tailwind | v0.dev로 채팅 UI 빠르게 |
| **Streaming** | Vercel AI SDK (`ai` 패키지) | 토큰 스트리밍 채팅 구현 표준 |
| **파일 출력** | `marked` → Markdown 변환은 사실 불필요 (이미 MD), 다운로드는 Blob → `<a download>` | 가벼움 |
| **배포** | Vercel | week-5와 동일 |
| **환경변수** | `.env.local` + Vercel 대시보드 | week-5 패턴 |

## 3. 데이터 모델 (Supabase)

```sql
-- 고객사 (라라에듀의 거래처)
create table public.client_companies (
  id bigint primary key generated by default as identity,
  name text not null,                    -- "삼성중공업 거제조선소"
  industry text,                          -- "조선/중공업"
  location text,                          -- "경남 거제"
  employee_count int,
  risk_factors text[],                    -- ["고소작업","용접","화학물질"]
  preferred_tone text,                    -- "보수적" / "혁신적"
  honorific text,                         -- "팀장님" / "리더"
  created_by uuid references auth.users(id),
  created_at timestamptz default now()
);

-- 콘텐츠 자산 (라라에듀가 보유한 것 — 시드 데이터)
create table public.content_assets (
  id bigint primary key generated by default as identity,
  category text not null,                 -- "산업안전" / "법정의무" / "요즘리더_AI"
  title text not null,
  body text not null,                     -- 강의 노트 원문
  source_url text,                        -- 원본 PPT/PDF 위치
  created_at timestamptz default now()
);

-- 생성 세션 (PM이 시작한 한 번의 대화)
create table public.generation_sessions (
  id bigint primary key generated by default as identity,
  client_company_id bigint references public.client_companies(id),
  user_id uuid references auth.users(id),
  content_category text,                  -- 어떤 카테고리 콘텐츠를 만드는 중
  status text default 'in_progress',      -- in_progress / completed / archived
  created_at timestamptz default now()
);

-- 채팅 메시지
create table public.session_messages (
  id bigint primary key generated by default as identity,
  session_id bigint references public.generation_sessions(id) on delete cascade,
  role text not null,                     -- 'user' / 'assistant' / 'system'
  content text not null,
  created_at timestamptz default now()
);

-- RLS: 라라에듀 직원 (auth.users 테이블에 소속된 사람) 모두가 모든 row 접근.
-- created_by 필드는 audit용. 외부 SaaS 판매 시점에 RLS 강화.
```

## 5. 위험 요소 (Pre-mortem)

| 위험 | 가능성 | 대응 |
|------|:-----:|------|
| Claude API 비용 폭주 (긴 컨텍스트 × 여러 고객사) | 중 | claude-haiku-4-5 메인 + 복잡한 건만 sonnet, 캐싱(prompt cache) 적용 |
| 라라에듀 콘텐츠 자산이 system prompt에 들어가면 길이 초과 | 중 | RAG로 단계 진행 — MVP는 5차시만, Phase 2에 벡터 DB |
| 라라에듀 직원이 안 씀 (변화 저항) | 고 | 김지원 PM과 7주차 시작 전 30분 인터뷰 → 워크플로우 정확히 매핑 |
| Supabase Free 티어 한계 (DB 0.5GB) | 저 | MVP는 충분. 1만건 메시지까지 OK |
| 토스페이먼츠 등 결제 모델은 Phase 2이지만 학습 동기 약화 | 저 | week-6 Q3·Q4에서 결제 학습이 별도로 진행됨 |

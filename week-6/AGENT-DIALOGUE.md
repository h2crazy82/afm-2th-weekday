# 에이전트 대화 — 결과물 개선 흔적 가이드

> 각 퀘스트의 "에이전트와 2회 이상 대화하며 결과물을 개선한 흔적" 스크린샷을 어떻게 만들지에 대한 안내.

## 🎯 평가자가 보고 싶어하는 "개선 흔적"이란?

**좋은 개선 흔적** (5pt 만점):
- 처음 만든 결과물과 마지막 결과물이 **다르다**는 증거
- AI가 질문을 던지고, 사용자가 답하고, AI가 다시 다듬는 **왕복 대화** 화면
- 막혔다가 해결한 흔적 (오류 → 수정 → 성공)

**약한 개선 흔적**:
- AI에게 한 번 시키고 결과물만 보여주는 화면
- 대화 없이 바로 코드만 보이는 화면

---

## 📸 이 세션에서 자동 캡처할 수 있는 것

이 한 세션 안에 **모든 4개 퀘스트의 개선 대화**가 들어있어. 다음 4개 모멘트를 캡처하면 4개 퀘스트 모두 커버 가능:

### 모멘트 1 — Q1 기획서 (요구사항 디테일 라운드)
**대화 흐름**: 사용자가 "디테일하게 확인하고 실행" 요청 → 에이전트가 R1~R7 7라운드 질문 던짐 → 사용자가 라운드별로 답변 → 에이전트가 요약하고 다음 라운드 진행
**캡처 추천 위치**: R3 (Q1 기획서 디테일) 또는 R7 답변 직후 "최종 실행 계획" 표가 보이는 화면

### 모멘트 2 — Q2 리서치 (Chrome MCP 사이트 자동 탐색)
**대화 흐름**: Docebo 네비게이션 첫 시도 실패 (newtab 그대로) → 에이전트가 wait 후 재확인 → 페이지 로드 성공 → 스크린샷 + get_page_text 자동 실행 → Sana Labs 쿠키 banner 만남 → 에이전트가 privacy-preserving 옵션 선택
**캡처 추천 위치**: Sana Labs 쿠키 처리 또는 Docebo 페이지 텍스트 스크롤 화면

### 모멘트 3 — Q3 빌드 에러 → 자동 수정
**대화 흐름**: `npm run build` 실행 → TypeScript 에러 ("Parameter 'cookiesToSet' implicitly has an 'any' type") → 에이전트가 즉시 4개 파일 (server.ts × 2, middleware.ts × 2) 수정 → 재빌드 → 16개 routes 빌드 성공
**캡처 추천 위치**: "Failed to compile" 에러 + 그 직후 Edit 호출 + 재빌드 성공 메시지 함께 보이는 화면

### 모멘트 4 — Q4 시드 5개 콘텐츠 자동 매핑
**대화 흐름**: 에이전트가 "콘텐츠 5개 시드 — Q1 MISSION의 라라에듀 콘텐츠 자산과 1:1 매핑되도록 작성" 결정 → seed.sql에 5개 진짜 프롬프트 자동 생성 → Supabase MCP로 INSERT → 5 rows 확인
**캡처 추천 위치**: seed.sql 5개 콘텐츠 결과 또는 Supabase MCP execute_sql로 5 rows 확인된 화면

## 🛠 어떻게 캡처할까

### 방법 A — Mac 단축키 (가장 쉬움)

각 모멘트가 보이는 Claude Code 터미널 화면에서:

```
cmd + shift + 4  → 영역 선택 → 마우스 드래그 → 자동으로 ~/Desktop 또는 클립보드에 저장
```

또는 창 전체:
```
cmd + shift + 4 → 스페이스바 → 창 클릭 → 창 전체 캡처
```

### 방법 B — 자동 (이 세션 종료 전 한 번에)

**스크롤백을 활용**: Claude Code 터미널을 위로 스크롤해서 위 4개 모멘트 각각 보이게 한 다음 캡처. 한 세션 안에 모두 있으니 스크롤만 하면 됨.

### 방법 C — 자동화 스크립트 (이 세션에서 실행)

Claude Code 창이 활성 상태일 때 다음 명령으로 화면 캡처:

```bash
osascript -e 'tell application "iTerm" to activate'   # 또는 Terminal/Cursor
sleep 1
screencapture -x ~/afm-2th-weekday/week-6/project_plan/screenshots/01-ai-planning-chat.png
```

> 단점: 캡처 시점에 보이는 화면만 잡힘. 4개 모멘트를 보이도록 매번 스크롤해야 함.

## ✨ 추가 팁 — 메인 캡처 1장 + 보조 캡처

평가자가 "스크린샷 한 장만 봐도 개선 흔적이 명확한 것"을 좋아함.

**가장 임팩트 있는 한 장**: 모멘트 3 (Q3 빌드 에러 → 자동 수정 → 성공). 이유:
- 첫 시도 실패가 명확히 보임
- 자동 수정 과정 (Edit 호출 4번) 가시화
- 재빌드 성공 (`✓ Compiled successfully` + 16 routes 표) 함께 보임
- "AI와 협업하며 결과물 개선"의 교과서적 예시

이 화면 한 장을 4개 폴더 모두에 동일하게 복사해도 무방. 단, 파일명을 폴더별로 다르게.

## 🤝 사용자가 직접 해야 하는 이유

내가 직접 스크린샷 자동 저장이 가능하지만:
- 캡처 시점에 어느 부분이 보이는지는 **사용자 터미널 스크롤 위치에 달림** → 사용자가 직접 스크롤한 뒤 캡처하는 게 가장 확실
- 4개 모멘트는 한 세션 내에 모두 있어서 **세션이 살아있는 동안** 캡처 추천

그래도 자동 시도 가능: 사용자가 "스크린샷 자동 캡처해줘"라고 하면 현재 활성 창을 스냅 → 해당 폴더에 저장.

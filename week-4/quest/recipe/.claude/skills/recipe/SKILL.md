# recipe

냉장고에 있는 재료로 1인분, 15분 이내, 자취생 난이도 레시피를 생성하고 마크다운으로 저장한다.

## 언제 사용하나

- "레시피 추천해줘" / "뭐 해먹지" / "/recipe"
- 15분 안에 만들 수 있는 1인분 메뉴 요청
- 자취생도 쉽게 따라할 수 있는 메뉴 요청

여러 날짜 식단 계획, 장보기 리스트, `ingredients/`에 없는 재료가 필요한 요리에는 사용하지 마라.

## 입력

- `ingredients/*.json` — 한 파일당 재료 하나. 모든 파일을 빠짐없이 읽는다. 각 파일은 `name`, `quantity`, `unit`, `category`, `expiresAt` 등을 포함할 수 있으며 존재하는 필드만 사용한다.

## 하드 제약

1. **인분**: 정확히 1인분
2. **조리 시간**: 준비 + 조리 합쳐서 15분 이내
3. **난이도**: 자취생 난이도 — 팬 하나 또는 냄비 하나, 특수 장비 없음, 복잡한 칼질/기법 없음. 소금·후추·식용유는 상비 조미료로 가정한다.
4. **재료**: `ingredients/*.json`에 있는 재료 + 위 상비 조미료만 사용. 재료를 지어내지 마라.
5. **칼로리**: 완성된 1인분의 예상 총 kcal을 반드시 포함한다 (대략적인 수치여도 무방, "예상 칼로리"로 명시).

## 실행 순서

1. `ingredients/*.json`을 glob으로 모두 찾아 Read 툴로 읽는다.
2. 각 파일의 재료와 수량을 정리한다. `expiresAt`이 임박한 재료가 있으면 우선 사용한다.
3. 제약을 만족하는 요리를 하나 고른다. 가능한 조합이 여러 개면 단계 수가 가장 적은 것을 택한다.
4. 실제로 사용한 재료의 대략적 kcal을 합산해 1인분 예상 칼로리를 계산한다.
5. `recipes/` 폴더가 없으면 만든다. `recipes/YYYY-MM-DD_<요리명>.md` 형식으로 저장한다 (오늘 날짜 사용). 같은 파일이 이미 있으면 덮어쓰지 말고 `_2`, `_3` 등 숫자 접미사를 붙인다.

## 저장할 마크다운 형식

```markdown
# <요리 이름>

- **조리 시간**: <N>분 (준비 <a>분 + 조리 <b>분)
- **난이도**: ★☆☆ 자취생 난이도
- **1인분 예상 칼로리**: 약 <N> kcal

## 재료 (1인분)
- <재료1> <수량>
- <재료2> <수량>
...

## 조리 순서
1. <단계 1>
2. <단계 2>
...

## 팁
- <선택: 맛/보관/대체 재료 팁 1~3개>
```

## 저장 후 응답

사용자에게는 짧게 아래 세 가지만 전달한다. 전체 레시피 본문은 저장 파일에 있으므로 채팅에서는 반복하지 않는다.
- 요리 이름
- 저장된 파일 경로
- 이 요리를 고른 한 줄 이유 (예: "김치 유통기한이 임박해서 선택")

## 에러 케이스

- `ingredients/` 폴더가 없거나 JSON이 하나도 없으면: "재료 파일이 없습니다. `ingredients/` 폴더에 재료 JSON을 추가해 주세요."라고만 답하고 파일을 만들지 않는다.
- 어떤 조합으로도 15분·1인분 제약을 만족할 수 없으면: 어떤 재료가 부족해서 불가능한지 한 줄로 알려주고 저장하지 않는다.
